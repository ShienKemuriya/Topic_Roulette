# REALITY話題ルーレット (Public版) ツールマニュアル

このドキュメントは、「REALITY 話題ルーレット (Public版)」の仕様、機能、および利用方法をまとめたマニュアルです。
Notebook LMやGeminiなどのLLMに読み込ませて、使い方を共有したり、今後のカスタマイズ案を相談するためのコンテキストとして活用できます。

## 概要
REALITYなどのライブ配信で利用できる「話題ルーレット」Webアプリケーションです。
ブラウザ上にソースとして取り込む（OBSなどの配信ソフトでウィンドウキャプチャやブラウザソースとして表示する）ことを前提に作られています。
プログラミングの知識がないユーザーでも簡単に自身の話題リストを管理・編集・共有できるように、GUIベースのエディタ機能とCSVのインポート/エクスポート機能を搭載しています。

## ファイル構成
- `index.html`: ルーレット本体のメイン画面。配信に映す画面です。
- `edit.html`: 話題の追加・編集・削除、およびCSVの読み書きを行う管理画面。
- `list.html`: 現在登録されている話題とクリア状況を一覧で確認できる画面。
- `style.css`: アプリ全体の共通デザインスタイルシート（サイバー・システム風デザイン）。
- `script.js`: ルーレットの回転アニメーション、状態管理を司るJavaScript。
- `data.js`: 初期データ（デフォルトの話題リスト）が定義されたファイル。

## 各画面の機能詳細

### 1. メイン画面 (`index.html`)
- **ルーレット機能**: 「SPIN ROULETTE」ボタンを押すと、未消化の話題からランダムで1つが選出されます。
- **重複排除**: 一度出た話題は「CLEARED（クリア済）」状態となり、以降のルーレット候補から自動的に除外されます。
- **リスト連携表示**: 画面右側の「Topic Database」枠内には、**すでに選ばれた（クリア済）話題だけ**がリストアップされます。
- **個別復元機能**: 右側リストに表示された各話題の「候補に戻す」ボタンを押すことで、そのトピックだけを再び未クリア状態（ルーレット候補）に戻すことができます。
- **全リセット**: 「RESET」ボタンで、すでに出たすべての話題のクリア状況を初期化し、すべてをルーレット候補に戻します。

### 2. 管理（エディタ）画面 (`edit.html`)
- **GUIでの話題編集**: ブラウザ上で直接「タイトル」と「詳細」をテキストボックスから入力して編集できます。
- **話題の追加・削除**: 「+ 新しい話題を追加」や各行の「削除」ボタンでリスト構成を直感的に変更できます。
- **クリア状態の操作**: すでにルーレットで出た話題は（※使用済み・クリア済）と表記され、「未クリアに戻す」ボタンで個別に状態を戻せます。
- **変更の保存**: 「💾 変更を保存する」ボタンを押すと、ブラウザのローカルストレージに最新リストが記録され、ルーレット本体画面に反映されます。

### 3. CSVバックアップとシェア機能 (`edit.html`内)
- **CSVエクスポート**: 現在の話題リスト（タイトル、詳細、クリア状態）を `.csv` 形式のファイルとしてローカルにダウンロードできます。これにより、話題リストのバックアップや、作成したリストの他ユーザーへの配布が可能です。
- **CSVインポート**: 他人が作成した（あるいはバックアップした）CSVファイルを読み込み、現在の話題リストを一括で上書き・更新することができます。
- ※CSVのフォーマットは「タイトル」「詳細」「クリア状態(TRUE/FALSE)」の3列のシンプルな形式です。Excel等の表計算ソフトで編集可能です。

### 4. 全話題一覧画面 (`list.html`)
- 現在登録されている全話題を、クリア状況のバッジと共に一覧で確認できます。全体の構成をざっと見渡したい時に利用します。

## 仕様上のポイント（開発者/AI向け情報）
- **状態管理 (State Management)**: 
  データはブラウザの `localStorage` (キー名: `reality_roulette_public_state`) を使用して永続化されています。そのためブラウザのリロードを行ってもクリア状況や編集した話題は失われません。
- **依存関係の排除**: 
  フレームワーク（ReactやVue）や外部ライブラリ（PapaParseなど）は一切使用せず、Vanilla JavaScript (ES6) と生のHTML/CSSで構築されています。これによりローカル環境でもダブルクリックで手軽に動作します。
- **レスポンシブデザイン**: 
  CSS GridとFlexboxを活用し、ウィンドウサイズを小さくしても崩れにくい設計になっています（OBSで配置を調整しやすい作り）。
